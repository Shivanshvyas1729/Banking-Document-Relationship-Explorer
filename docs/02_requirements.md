# Functional Requirements: BankDocLink

This document defines the functional requirements and user stories for BankDocLink, categorized by milestone/phase.

---

## Phase 1: MVP UI & Document Upload
- **User Story 1**: As a user, I want to upload multiple files in PDF, DOCX, or TXT formats, so that I can analyze them together.
- **User Story 2**: As a user, I want to see a list of my uploaded documents showing their filename, type, and file size, so that I know what is currently in my session registry.
- **User Story 3**: As a user, I want to delete any uploaded document from the registry, so that it is removed from memory and local disk.
- **User Story 4**: As a user, I want the system to restrict me from uploading unsupported files or exceeding the maximum capacity limit (10 documents), showing a clear warning message.

---

## Phase 2 & 3: Document Parsing, NLP Entity Extraction, & Vector Storage
- **User Story 5**: As a system, I want to parse uploaded files into plain text:
  - TXT files should be decoded cleanly.
  - PDF files should be parsed into Markdown format (using `pymupdf4llm`).
  - DOCX files should be partitioned into clean text blocks.
- **User Story 6**: As a system, I want to pass parsed document text through a spaCy NLP pipeline to extract:
  - Key named entities (people, organizations, dates, money values).
  - Main topics/noun phrases.
  - Essential keywords/key terms.
- **User Story 7**: As a system, I want to split document text into chunks, generate high-quality text embeddings using a local Sentence-Transformers model, and index them into an in-memory vector database (FAISS/Chroma).

---

## Phase 4: RAG Query & Relationship Explanation
- **User Story 8**: As a user, I want to submit a natural language question about the documents, and receive a comprehensive, contextually accurate response generated by an LLM.
- **User Story 9**: As a user, I want the system to complete my queries in less than 10 seconds, falling back gracefully to a helpful warning if no relevant documents are found or if the API times out.

---

## Phase 5: Document Insights UI
- **User Story 10**: As a user, I want to click and expand any uploaded document in the UI to see its extracted entities, topics, and key terms presented in modern, clean visual chips.

---

## Phase 6: Relationship Visualization & Query UI
- **User Story 11**: As a user, I want to see an interactive network graph visualization showing how documents connect to shared entities and topics.
- **User Story 12**: As a user, I want to type query questions in a clean chat-like interface, see a loading spinner while the answer is generated, and see clear citations linking to the source documents.
